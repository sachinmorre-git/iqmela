# Project Context and Guidelines

We are building the "Interview Platform" (IQMela-v2) in very small, incremental steps.

## Important Rules
- **Do only the exact step I provide:** Keep changes restricted to the current objective.
- **Do not add extra features:** Only fulfill what is requested in the current step.
- **Visual & Testable:** The result of each step must be visible and testable in the browser.
- **Quality:** Keep the code production-minded and modular.
- **Stack:**
  - **TypeScript**
  - **Next.js App Router** (`app/` directory)
  - **Tailwind CSS**
  - **shadcn/ui** style components (where appropriate)
- **Architecture:** Use a clean, logical folder structure.
- **Independence:** Keep every step independently testable.
- **Reporting:** After every step, output exactly these 5 points:
  1. Files changed
  2. Commands to run
  3. URL/path to open (**must be a clickable hyperlink**)
  4. What should appear on screen
  5. How to validate success
